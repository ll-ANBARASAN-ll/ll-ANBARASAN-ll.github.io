Amplificateur & Écrêteur (SAé 1.01)

SAé 1.01 - IUT Créteil-Vitry

Objectifs du Projet

Concevoir une carte électronique capable de traiter un signal d'entrée sinusoïdal de faible amplitude (Ve) pour fournir deux sorties distinctes :
- Vs1 : Signal amplifié sinusoïdal.
- Vs2 : Signal écrêté symétriquement.

Caractéristiques Techniques

Alimentation :

+/- 15V Symétrique

Étage Amplificateur :
  • Montage Non-Inverseur à base de TL081 (ou LM308 en simu).
  • Gain |A| = 400 +/- 10%.
  • Impédance d'entrée : 35kΩ < Ze1 < 45kΩ.
Étage Écrêteur :
  • Écrêtage à diodes (1N4148) et résistances.
  • Sortie Vs2 = 14V crête à crête (pour Vs1 = 10V crête).
  • Impédance d'entrée Ze2 < 3kΩ.

Schéma Électronique

Réalisation & Outils

  • Simulation : LTspice pour valider les calculs théoriques.
  • Conception : Schématique et Routage PCB sous KiCad.
  • Composants : AOP TL081, Diodes 1N4148, Résistances.

Générateur de Signal (SAé 1.02)

SAé 1.02 - IUT Créteil-Vitry

Fonctionnement Général

Ce projet consiste à créer un générateur de signal triangulaire. Le système utilise une horloge pour incrémenter un compteur binaire. La sortie binaire est ensuite convertie en tension analogique via un réseau R-2R, créant ainsi une rampe (signal triangulaire).

Blocs Fonctionnels

1. Horloge (NE555) :

Génération d'un signal carré périodique (fréquence réglable via résistances RA, RB et condensateur C).

2. Compteur (CD4040) :

Compteur binaire asynchrone qui reçoit l'horloge et incrémente sa valeur sur ses sorties Q0 à Qn.

3. Convertisseur N/A (Réseau R-2R) :

Transforme le mot binaire du compteur en une tension proportionnelle. Utilisation de résistances de 100Ω et 200Ω.

4. Comparateur (LM311) :

Compare la tension de sortie à une référence pour réinitialiser le compteur et créer la forme triangulaire.

Station Météo Connectée (IoT)

SAé 4.ESE.01 - Technologies IoT - 2ème Année

Objectif

Conception d'un objet connecté permettant de mesurer la température et le taux d'humidité d'une pièce, puis de transmettre ces données via le réseau LoRaWAN vers un serveur cloud pour consultation sur application mobile.

Analyse Fonctionnelle (Bête à corne) :

  • À qui le produit rend-il service ? À l'utilisateur.
  • Sur quoi agit-il ? Sur l'environnement.
  • Dans quel but ? Connaître la température et le taux d'humidité en temps réel.

Architecture Matérielle & Schéma

  • Microcontrôleur : STM32 Nucleo-L432KC (Ultra-low-power).
  • Communication : Module Nemeus XM001 (LoRa 868 MHz / Sigfox / FSK).
  • Capteurs : SHT3x (Température & Humidité) via I2C.
  • Alimentation : Batterie Li-Ion, BMS et Régulateur 3V3 (ADP3338).
  • Interface : Switch pour activation manuelle.

Réseau LoRaWAN

  • Protocole : LoRaWAN Classe A (Optimisé pour l'énergie : réception uniquement après émission).
  • Infrastructure : Pico-Gateway MG003-L-EU (Relais vers serveur réseau IP).
  • Activation : OTAA (Over-The-Air Activation) pour une connexion sécurisée dynamique.

Programmation (STM32CubeIDE)

Développement en langage C avec exploration de différents modes pour optimiser la consommation :

  • Mode Pooling : Scrutation simple (boucle infinie), utile pour les tests initiaux mais énergivore.
  • Mode IT (Interruptions) : Le processeur ne réagit qu'aux événements (ex: réception UART), plus efficace.
  • Mode DMA (Direct Memory Access) : Transfert de données (ex: mesures capteurs vers mémoire) sans solliciter le CPU, idéal pour la basse consommation.
